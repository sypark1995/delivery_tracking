# 디자인 가이드라인

한국 거주 외국인 대상 택배 조회 앱 · Compose Multiplatform (Android + iOS)
**확정 방향: Light & editorial — 웜 뉴트럴 배경 + 그린 액센트 + 라이브 트래킹 히어로**

> 함께 보는 문서: `기능명세서.md`(기능·아키텍처), `로드맵.md`(단계별 계획). 이 문서의 화면·컴포넌트는 두 문서와 일치하도록 작성됨.
>
> **핵심 화면(8종)**: 온보딩·언어선택 / 홈·배송목록(+빈 상태) / 송장 추가 / 배송 상세 / 기사 연락 / 설정 / 알림·Updates.
> **확장 화면(5종, Phase 2~3)**: 공유 등록 시트 / 배송 사진(상세) / 위젯·라이브 액티비티 / 무인택배함·재배송 / 배대지 end-to-end. 전용 관세 계산 화면은 두지 않음(§4.1).

## 1. 디자인 방향과 원칙

밝고 따뜻한 배경 위에 큰 타이포와 단정한 카드, 그리고 하나의 그린 액센트로 신뢰감과 명료함을 준다. 상단의 **라이브 트래킹 히어로 카드**가 "진행 중인 내 배송"을 즉시 보여주는 것이 이 앱의 시그니처.

사용자는 한국어가 익숙하지 않고, 불안 없이 "내 물건 어디쯤"을 빠르게 확인하려는 외국인이다. 모든 결정은 아래에 우선한다.

1. **명료함 우선** — 상태를 색·아이콘·타임라인으로 먼저. 텍스트는 보조.
2. **신뢰감** — 물류/금융에 준하는 차분·단정. 과한 장식 지양.
3. **언어 중립** — 아이콘·색·레이아웃만으로도 대략 이해되게. 텍스트는 보조.
4. **불안 최소화** — 지연·미배송에도 다음 행동(재조회·기사 연락)을 함께 제시.

## 2. 디자인 기반 (기술)

- **Compose Multiplatform**으로 Android·iOS UI 공유. **스톡 Material 3 룩은 채택하지 않는다.** Material3 라이브러리는 기술적 뼈대(컴포넌트 골격)로만 쓰고, 색·타이포·모양·간격은 **자체 디자인 토큰**으로 완전히 커스텀한다.
- **토큰 기반**: 아래 색/타이포/간격/라운드를 하드코딩하지 않고 테마 토큰(`AppTheme`)으로 관리 → 다크모드·다국어·리브랜딩 대응.
- **플랫폼 적응**: 기본 룩은 양 플랫폼 통일. iOS는 HIG 관례 존중(네이티브 스크롤/바운스, 스와이프 뒤로가기, 세이프에어리어, 최소 탭 타깃 44pt).

## 4. 홈(메인) 정보 위계 — 확정

이 앱은 배송을 등록해두고 반복 확인하는 개인 트래커다. 따라서 **목록(내 배송) 우선, 조회/추가는 상시 진입점**으로 둔다. 조회창을 메인 전체로 깔지 않는다.

- **등록건 있음(기본)**: `상단 헤더 → 조회/추가 진입점 → 세그먼트 필터(Active/Delivered/All) → 라이브 히어로(진행중) → 최근 목록`.
  조회/추가는 **상단 고정 "Add tracking number" 바** 또는 우하단 + 버튼으로 항상 노출.
- **첫 실행·빈 상태**: 보여줄 목록이 없으므로 **조회/추가 입력을 화면 중앙에 크게** + 짧은 안내. 이때만 "조회 먼저" 형태로 전환.

### 4.1 통관·관세 처리 (확정)

전용 관세 계산 화면은 두지 않는다. 사용자가 추적 시점에 실제로 원하는 것은 세금 계산이 아니라 ① 통관에 걸렸는지(상태), ② 개인통관고유부호를 꺼내 쓰는 것이다. 대부분의 추적 대상은 국내 택배라 관세와 무관하고, 부정확한 세금 숫자는 신뢰를 훼손한다.

- **통관 상태**: 배송 상세 타임라인·알림에 번역된 한 줄("Customs clearance in progress")로 녹인다. 색은 이동중(블루) 톤 재사용 + 라벨로 구분.
- **개인통관고유부호(P번호)**: 설정에서 저장·복사(마스킹 표시).
- **예상 세금**: 필요 시 후순위 "참고용" 기능으로만. 코어 화면 아님.

## 5. 핵심 컴포넌트

- **라이브 트래킹 히어로 카드**: 진행 중 배송 1건을 크게. `상태 뱃지 + 큰 ETA + 상품·택배사 + 진행바 + 기사 전화 CTA`. 카드 톤은 상태 색, CTA는 브랜드 그린.
- **배송 목록 카드(흰 카드+보더)**: 상품명 + 택배사(secondary) + 우측 **상태 태그 pill**. 탭 → 상세.
- **상태 태그/뱃지**: tint 배경 + 아이콘 + 번역 라벨. 색만으로 구분하지 않음.
- **세그먼트 필터**: Active / Delivered / All. 활성 세그먼트 브랜드 그린.
- **조회/추가 바**: 상단 고정. placeholder "Add tracking number", 붙여넣기·스캔 아이콘.
- **배송 타임라인(상세)**: 세로 스텝. 완료=채워진 노드, 현재=강조, 예정=흐림. 각 스텝 번역 상태 + 시각 + 위치. 원문(한글) 토글 제공.
- **기사 연락 CTA**: 배송중일 때만. 큰 전화 버튼(브랜드 그린) + 기사명. 번호 없으면 숨김(폴백).
- **기사 메시지 브릿지**: 사용자가 자국어로 입력한 문장을 **한국어로 자동 번역**해 전송/복사(예: "Please leave it at the door" → "문 앞에 놓아주세요"). 원문·번역문 함께 표시.
- **알림 · Updates 화면**: 상단 최신 푸시 카드 + 지난 업데이트 목록. 상태 변경·통관 진행을 번역해 한 줄로. (실시간 아님 — 백그라운드 페치 기반, `기능명세서 §3.10`.)
- **빈 상태/에러**: 일러스트 + 다음 행동 버튼. 부정 상태도 차분하게.

### 5.1 확장 기능 컴포넌트

- **홈 위젯 · 라이브 액티비티**: 진행중 배송을 축약 표시(상태 아이콘 + ETA + 미니 진행바). 잠금화면·홈화면·다이나믹 아일랜드 크기별 레이아웃. 브랜드 그린 미니멀. 활성 배송 없으면 "등록된 배송 없음" 상태.
- **배송 완료 사진 카드(상세)**: 배송완료 시 현관 사진을 라운드 이미지로. 없으면 섹션 숨김(폴백). 탭 → 확대.
- **무인택배함/재배송 화면**: 부재중 카드 + 재배송 버튼(딥링크, 그린) + 지도 미리보기 + 가까운 수령처 리스트(거리순).
- **공유 등록 시트**: 시스템 공유 시트에서 뜨는 축약 카드. 감지된 송장번호 + 택배사 + "추가" 버튼. 앱 진입 없이 등록.
- **배대지 구간 타임라인**: 상세 타임라인에 구간 헤더(Overseas / Customs / Domestic)로 그룹핑. 구간별 색·라벨로 여정 단계 구분.

## 6. 다국어(i18n) 디자인 규칙

- **텍스트 확장 대비**: 언어별 길이 상이(베/러 길어짐). 버튼·뱃지 고정폭 금지, 2줄까지 자연스럽게.
- **문자열 조립 금지**: 자리표시자 포함 완성형 문자열을 언어별로 통째 번역(`{count}개 남음`).
- **숫자·날짜·통화 현지화**: 로케일 포맷 사용.
- **이미지 안 텍스트 금지**: 레이어 분리.
- **RTL 대비**: `start/end` 기준 레이아웃(좌우 하드코딩 금지). 초기 언어(영/중/베/한)엔 RTL 없지만 아랍어 확장 대비.
- **언어 전환**: 온보딩 첫 화면 + 설정에서 항상. 기기 로케일 자동 감지 후 수동 오버라이드.
- **번역 품질 안전장치**: API 상태 메시지 기계번역엔 원문(한글) 토글로 대조 가능하게. (고정 용어는 자체 사전 우선 — `기능명세서 §2` 참조.)

## 7. 아이콘 · 모션 · 접근성

- **아이콘**: 아웃라인 통일(Tabler / Material Symbols Outlined). 상태 아이콘은 §3.3 표로 고정.
- **모션**: 절제. 상태 변경 시 부드러운 전환, 현재 스텝만 은은한 강조(진행바 채움). 과한 애니메이션 금지.
- **접근성**: 대비 AA 이상, 색+아이콘+텍스트 3중 표시, 스크린리더 라벨(VoiceOver/TalkBack), 동적 글자 크기 지원. CMP는 iOS VoiceOver·풀 키보드 접근 기본 지원.

## 10. 하지 말 것 (안티패턴)

- 색만으로 상태 구분(색맹 배제).
- 번역 텍스트를 이미지에 삽입.
- 문자열을 코드에서 조각조각 연결.
- 고정폭 버튼에 긴 번역어 → 잘림.
- 메인 전체를 조회 입력창으로(반복 사용 불리) — 목록 우선 + 조회는 상단 고정.
- "실시간" 표기 후 실제 15분 폴링(신뢰 훼손) — `기능명세서 §3.10` 참조.
- 부정 상태를 경고만 던지고 다음 행동 미제시.
- 관세 계산기를 코어 화면으로(추적 시점 니즈와 어긋남) — 통관 상태 표시 + P번호 관리로 대체(§4.1).
