# 개발 로드맵

한국 거주 외국인 대상 택배 조회 앱 · KMP/CMP · 백엔드 없음

---

## Phase 0 — 기반 셋업 (1~2주)

KMP/CMP 프로젝트를 세우고 뼈대를 만드는 단계.

- KMP + Compose Multiplatform 프로젝트 구성 (androidApp / iosApp / shared)
- Ktor 클라이언트, SQLDelight, DI(Koin) 셋업
- i18n 리소스 구조 + 기기 로케일 감지 (영/중/베/한 4개 언어로 시작)
- CI에서 안드로이드·iOS 빌드 확인

**완료 기준**: 두 플랫폼에서 빈 앱이 같은 코드로 실행됨.

---

## Phase 1 — MVP (3~5주)

"송장 넣으면 번역된 상태가 보인다"까지. 단일 API 키를 앱에 넣고(난독화) 출시.

- 송장 추가 (수동 입력 + 붙여넣기)
- 택배사 자동 감지 + 수동 선택 폴백
- 실시간 배송조회 + 시간순 타임라인
- **상태 메시지 번역** (고정 용어 자체 사전 우선)
- 통합 배송 목록 + 상태 요약 뱃지
- 로컬 저장 (송장 목록·이력)
- 다국어 UI (4개 언어)

**완료 기준**: 안드로이드·iOS에서 다국어로 통합 조회·번역이 동작.

---

## Phase 2 — 외국인 특화 확장 (4~6주)

차별화 기능을 얹는 단계. 여기서 경쟁 우위가 생김.

- **배송기사 연락** (스윗트래커 `telno`, 배송중 시 전화·번역 스크립트, 폴백 UI)
- 주소 영↔한 변환·검증 (도로명주소 API)
- 통관 상태 표시(상세·알림) + 개인통관고유부호(P번호) 저장·복사 (전용 관세 계산기는 코어 제외)
- 온디바이스 번역 확장 (자유 텍스트, ML Kit/Apple Translation expect-actual)
- 주문 메일 파싱 → 송장 자동 등록
- **공유 익스텐션 자동 등록** — 브라우저·쇼핑앱·메일에서 "공유 → 앱"으로 송장 등록 (④)
- **배송 완료 사진 표시** — API가 사진 URL을 주면 상세에 노출, 없으면 숨김 (②)
- 상태 변경 로컬 알림 (백그라운드 페치)
- 언어 확장 (태국어·러시아어·일본어 등)

**완료 기준**: 기사 연락·주소·통관까지 외국인 핵심 페인포인트 커버.

---

## Phase 3 — 확장·파워 기능 (무백엔드, 5~7주)

서버 없이 가능한 고부가 기능. 여기서 "쓸 만한 앱"에서 "계속 쓰는 앱"으로.

- **홈 위젯 · 라이브 액티비티** (①) — iOS WidgetKit + ActivityKit / 안드 Glance + 상시 알림. 진행중 배송을 잠금·홈화면에. 갱신은 백그라운드 페치 기반(실시간 아님)
- **무인택배함/편의점 수령 + 재배송** (③) — 위치 지도 안내 + 재배송은 택배사 앱/웹 딥링크. 부재중 상태에서 진입
- **배대지 end-to-end 추적** (⑤) — 해외 발송→배대지→국제운송→통관→국내택배를 조회 API로 이어붙인 단일 타임라인
- 태그/그룹(가족·쇼핑몰별) · 검색 · 정렬
- 여러 배송지 관리 · 이력 CSV 내보내기
- 지연·분실 이상 감지("N일째 정지") · 일일 요약 다이제스트 · 방해금지
- 오프라인 캐시(마지막 상태 보기)
- Apple Watch / Wear OS 요약
- 프리미엄 인앱결제(무제한 송장·추가 언어·광고 제거) — 스토어 결제라 서버 불필요

**완료 기준**: 위젯·배대지·무인함까지 반복 사용성과 편의가 완성됨.

---

## 범위 밖 — 서버가 있어야 가능 (현재 제외)

아래는 "백엔드 없음" 원칙상 현재 범위에서 제외. 훗날 서버리스 함수를 두게 되면 검토.

| 기능 | 필요 이유 |
|------|-----------|
| API 키 완전 은닉 | 프록시 서버 필요(현재는 난독화 + 쿼터 알림으로 감수) |
| 진짜 실시간 즉시 푸시 | Webhook 수신 서버 + FCM/APNs |
| 기기 간 동기화 | 계정 + 원격 DB (대안: iCloud/Drive 백업으로 부분 대체 가능) |

핵심: 지금은 전부 클라이언트로 구현. 서버는 필요가 확실해질 때 **함수 하나부터** 점진 도입.
