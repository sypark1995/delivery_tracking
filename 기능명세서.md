# 택배 조회 앱 기능 명세서

한국 거주 외국인 대상 · 백엔드 없는 순수 클라이언트 모바일 앱

---

## 1. 기술 아키텍처

### 스택
- **Kotlin Multiplatform (KMP)** — 비즈니스 로직 공유 (API 호출, 응답 파싱, 택배사 감지, 관세 계산, 로컬 DB, i18n 로직). KMP는 2023년 11월 stable, 넷플릭스·맥도날드·캐시앱 등 프로덕션 검증됨.
- **Compose Multiplatform (CMP)** — UI 공유. iOS는 1.8.0(2025.05)부터 정식 stable, 현재 1.11.0. 안드로이드·iOS UI를 한 코드베이스로.
- **Ktor Client** — 공유 네트워킹 계층에서 택배·주소 API 직접 호출.
- **SQLDelight** — 기기 로컬 DB (송장 목록, 배송 이력, 통관고유부호).
- **백엔드 없음** — 모든 API는 앱에서 on-demand 직접 호출. 서버·DB 운영 없음.

### 모듈 구성 (권장)
```
shared/
 ├─ commonMain     공통 로직·UI (대부분 여기)
 │   ├─ data       API 클라이언트, DTO, 파서, 로컬 DB
 │   ├─ domain     택배사 감지, 관세 계산, 상태 매핑
 │   ├─ ui         Compose 화면 (홈/상세/추가/설정)
 │   └─ i18n       언어 리소스, 용어 사전
 ├─ androidMain    ML Kit 번역 actual, 백그라운드 페치
 └─ iosMain        Apple Translation actual, BGTaskScheduler
androidApp/        진입점
iosApp/            진입점 (SwiftUI 래핑)
```

### 백엔드 없음에 따른 설계 원칙
- 실시간 Webhook 푸시 불가 → OS 백그라운드 페치로 주기적 조회 후 **로컬 알림** (근사치).
- API 키 완전 은닉 불가 → MVP는 난독화 + 쿼터 알림으로 감수. 필요 시 서버리스 함수 1개로 승격.
- 데이터는 기기 로컬에만 저장 → 기기 간 동기화 없음(대신 프라이버시 이점).

## 2. 다국어(i18n) 전략 — 2계층

| 계층 | 대상 | 방식 |
|------|------|------|
| **정적 UI 문자열** | 버튼·라벨·안내문 | Compose Multiplatform 리소스(`composeResources/values-xx/`)로 언어팩 번들. 기기 로케일 자동 감지 + 수동 전환. |
| **동적 API 텍스트** | 배송상태 메시지(한글로 옴) | ① 고정 용어(집화·간선상차·배송출발 등)는 **자체 매핑 사전**으로 정확·오프라인 번역. ② 자유 텍스트(비고 등)는 온디바이스 번역. |

### 온디바이스 번역 (expect/actual)
```kotlin
expect class Translator {
    suspend fun translate(text: String, target: Lang): String
}
// androidMain → Google ML Kit On-Device Translation
// iosMain     → Apple Translation framework (iOS 17.4+)
```
- 고정 용어 사전이 우선. 사전에 없는 문자열만 번역기 호출 → 정확도·속도·비용 최적.
- 지원 언어(초기): 영어, 중국어(간체), 베트남어, 한국어 + 확장(태국어·러시아어·일본어).

## 3. 기능별 명세

### 3.1 배송 조회 (핵심)
- 송장번호 입력 → 실시간 조회. Delivery Tracker `Query.track` 또는 스윗트래커 조회 API를 on-demand 호출.
- 응답의 이벤트 타임라인을 시간순으로 표시, 각 상태를 사용자 언어로 번역.
- 상태 요약 뱃지: 접수 / 이동중 / 배송출발 / 배송완료 / 미배송.

### 3.2 택배사 자동 감지
- 송장번호 텍스트 입력 시 택배사 추론(Delivery Tracker `searchText`). 후보 리스트 제시 + 수동 선택 폴백.

### 3.3 통합 조회
- 여러 택배사 송장을 한 목록/한 화면에서. 택배사별 상태 정규화(공통 status enum으로 매핑).

### 3.4 상태 메시지 번역 (외국인 특화)
- 위 i18n 2계층 적용. 택배 전문용어 내장 사전 + 온디바이스 번역.
- 용어 툴팁: "간선상차 = 물류센터 간 이동" 식 설명 제공.

### 3.5 배송기사 연락 (배송중 시)
- **스윗트래커 API 사용 시** 응답 `trackingDetails`에 `manName`(기사명)·`telno`(연락처)가 포함됨. 배송출발/배송중 단계에서 노출.
- 전화걸기 버튼 + "기사에게 보낼 메시지"를 한글로 번역해 문자 스크립트 제공.
- **폴백 필수**: 택배사·정책에 따라 번호가 null·마스킹될 수 있음 → 있으면 버튼 노출, 없으면 숨김. (Delivery Tracker는 기사 번호 미제공.)

### 3.6 주소 영↔한 변환·검증
- 정부 도로명주소 API(juso.go.kr) on-demand 호출. 영문주소 → 한글 도로명 + 우편번호 자동.
- 키는 패키지 서명 제한 지원(부분적 보안 이점).

### 3.7 통관 상태 · 개인통관고유부호 (관세 계산기 제외)
- **전용 관세 계산기는 코어에서 제외.** 대부분 국내 택배라 무관하고, 부정확한 세금 숫자는 신뢰를 훼손함. (디자인가이드라인 §4.1)
- 통관 진행 상태를 배송 상세 타임라인·알림에 번역해 한 줄로 노출("통관 진행중" 등).
- 개인통관고유부호(P번호) 로컬 저장·복사(설정, 마스킹 표시).
- 예상 세금은 필요 시 후순위 "참고용" 기능으로만.

### 3.8 주문 메일 파싱
- 쿠팡·알리·아마존 주문확인 메일 텍스트 붙여넣기 → 정규식으로 송장번호 자동 추출·등록.

### 3.9 로컬 데이터
- 송장 목록·배송 이력·통관번호를 SQLDelight로 기기 저장.
- 받은 택배 이력 대시보드(월별 개수·판매처).

### 3.10 알림 (제약 있음)
- OS 백그라운드 페치(안드 WorkManager / iOS BGTaskScheduler, 약 15분 주기·보장 안 됨)로 등록 송장을 주기 조회 → 상태 변경 시 **로컬 알림**.
- 실시간 즉시 푸시 아님. 폴링 최소화(활성 배송건만, 완료건 제외).

### 3.11 홈 위젯 · 라이브 액티비티 (①)
- **iOS**: WidgetKit 홈/잠금화면 위젯 + ActivityKit 라이브 액티비티(진행중 배송의 상태·ETA·진행바). **안드로이드**: Glance 홈 위젯 + 진행중 배송 상시(ongoing) 알림.
- 표시: 다음/진행중 배송 1건의 상태·ETA·진행률. 탭 → 상세.
- 갱신: 앱 실행 중 + 백그라운드 페치 시. **푸시 없이라 실시간은 아님**(라이브 액티비티도 제한적 갱신). 활성 배송 없으면 위젯은 "등록된 배송 없음" 상태.

### 3.12 배송 완료 사진 (②)
- 배송 상세에 **배송 완료 사진(현관 등)** 표시 — 조회 API 응답에 이미지 URL이 포함될 때만. 탭 → 확대.
- **폴백**: 사진 없으면 섹션 자체를 숨김(택배사·정책에 따라 미제공 흔함).
- 이미지 로컬 캐시, 프라이버시상 외부 전송 없음.

### 3.13 무인택배함/편의점 수령 · 재배송 (③)
- 부재중/미배송 상태에서 상세·알림에서 진입.
- **수령 위치 안내**: 무인택배함·편의점(GS Postbox·CU 등) 위치를 지도 API로 표시(가까운 순).
- **재배송 신청**: 앱 내 접수 불가가 일반적이므로 택배사 앱/웹으로 **딥링크**. 딥링크 불가 시 안내 + 전화.

### 3.14 공유 익스텐션 자동 등록 (④)
- iOS Share Extension / Android Share target 등록. 브라우저·쇼핑앱·메일에서 **텍스트·URL 공유 → 앱**.
- 공유된 내용에서 송장번호 정규식 추출 → 택배사 자동 감지 → 등록 확인 시트. 앱 진입 없이 빠른 추가.

### 3.15 배대지(배송대행지) end-to-end 추적 (⑤)
- 하나의 배송을 여러 구간으로 이어붙여 단일 타임라인: **해외 발송(해외 캐리어) → 배대지 입고 → 국제운송 → 통관 → 국내택배**.
- 구간 전환 시 운송장 승계(배대지 제공 국내송장 또는 사용자 입력). 각 구간은 조회 API **온디맨드 호출**이라 서버 불필요.
- UI: 상세 타임라인에 구간 헤더(해외/통관/국내)로 구분. 국제 구간은 커버리지에 따라 표시 한계 있을 수 있음(폴백).

## 4. API 선택 요약

| 요구 | 권장 |
|------|------|
| 기사 연락처 필요 | **스윗트래커(스마트택배)** — `telno` 제공 |
| 영어 문서·오픈소스·MVP 접근성 | Delivery Tracker (단, 기사 번호 없음 / 무료 키 21일 만료) |
| 주소 변환 | 도로명주소 API(juso.go.kr) |
| 번역 | ML Kit(안드) / Apple Translation(iOS) + 자체 용어 사전 |

## 5. 제약·리스크

- **API 키 노출**: 클라이언트에 심는 이상 추출 가능. MVP 감수 → 서버리스 승격 옵션.
- **실시간성**: 백그라운드 페치는 지연·미실행 가능. "실시간"으로 과장 금지.
- **기사 번호 가용성**: 택배사 정책 의존. 방어적 UI 필수.
- **Delivery Tracker 무료 키 21일 만료**: 실배포 시 Pro($49.99/월) 또는 스윗트래커 고려.
- **iOS 온디바이스 번역**: Apple Translation은 iOS 17.4+ 필요 → 하위 버전 폴백 설계.
